## Успешно проверено

- Вход с предоставленными тестовыми данными открывает дашборд.
- Работают редиректы:
  - `/auth` → `/auth/login`;
  - `/auth/create-company` → `/auth/register`;
  - `/structure` → `/employees`.
- Неизвестный обычный URL показывает 404 со ссылкой «На главную».
- `/share/article/fake` показывает состояние «Статья недоступна».
- Невалидный invite-token показывает состояние «Приглашение недействительно».
- Создание сотрудника обновляет список без F5.
- Крайние пробелы в имени и фамилии сотрудника обрезаются.
- Имя из одних пробелов отклоняется, заполненные данные остаются в модалке.
- Поиск сотрудников устойчив к `(`, `[`, `*`, `\\` и `<script>alert(1)</script>`.
- Поиск без результатов показывает отдельный текст «Никого не нашлось».
- Мобильная модалка сотрудника:
  - помещается в 375×812;
  - блокирует прокрутку фона;
  - имеет `role="dialog"` и `aria-labelledby`.
- Во время пройденной навигации ошибок и warning-ов в browser console нет.

## Ограничения аудита

- Принудительные ошибки API не проверялись: опубликованный стенд не позволяет менять `ERROR_RATE`.
- Полные CRUD-циклы БЗ, Академии, Графика и Распределения не завершены.
- Не выполнялись нагрузочный прогон и кроссбраузерная проверка.
- Network log недоступен через использованный браузерный интерфейс; выводы основаны на UI, DOM, консоли и устойчивой воспроизводимости.
