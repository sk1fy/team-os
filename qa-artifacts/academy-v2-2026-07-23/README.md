# Frontend QA — Академия V2

Дата проверки: 23.07.2026
Стенд: `http://31.76.42.128:8080`
Роль тестового пользователя: владелец
Desktop viewport: 1280×720
Mobile viewport: 375×812

## Итог

- Пройдено без дефектов: общий контейнер, навигация владельца, главная обучения,
  каталог и его поиск, список курсов и валидация формы создания, шаблоны,
  партнёрские курсы, внутренний отчёт и его URL-фильтры, состояние 404 курса,
  legacy-редиректы, desktop/mobile layout.
- Обнаружено: 4 уникальных блокирующих дефекта.
- Заблокировано дефектами: создание и конструктор курса, прохождение курса,
  подробный отчёт enrollment, внешний реестр, публичное внешнее обучение.
- Не проверено из-за отсутствия тестовых аккаунтов: матрица прав сотрудника и партнёра.
- Не проверено из-за отсутствия валидного внешнего токена: успешная активация,
  прохождение и завершение внешнего курса.

## Результат исправления и повторной проверки

Все четыре блокирующих дефекта исправлены и повторно проверены на локальной
сборке с подключением к API стенда:

- создание курса работает в HTTP-контексте и открывает созданный черновик;
- невалидная публичная ссылка показывает штатный экран «Ссылка недоступна»;
- подробный отчёт прохождения открывается и показывает уроки;
- внутренний плеер загружает структуру курса, текущий урок и прогресс;
- «Внешние ученики» открываются без Error Boundary и корректно показывают
  пустой результат.

Во время регрессии дополнительно найдены и исправлены два блокера конструктора:
зависимость от data-router при использовании `useBlocker` и несовпадение полей
черновика/прав курса с фактическим API. После исправления конструктор показывает
название курса, версию, разделы и пустое состояние урока.

## QA-ACADEMY-001 — `crypto.randomUUID()` ломает критические действия на HTTP-стенде

Приоритет: P0 / blocker
Область: создание курса, публичное обучение и другие mutation-flow

### Шаги: создание курса

1. Авторизоваться владельцем.
2. Открыть `/academy/courses`.
3. Нажать «Создать курс».
4. Выбрать «С нуля».
5. Ввести корректное название.
6. Нажать «Создать и открыть конструктор».

### Фактический результат

Показывается toast «Не удалось создать курс». Курс не создаётся, конструктор не
открывается.

### Ожидаемый результат

Создаётся draft курса, пользователь переходит в конструктор.

### Шаги: публичное обучение

1. Открыть `/training/:token` на текущем HTTP-стенде.

### Фактический результат

Экран полностью падает с ошибкой `crypto.randomUUID is not a function`.

### Ожидаемый результат

Для невалидного токена показывается штатное состояние недействительной ссылки.
Для валидного токена открывается landing внешнего курса.

### Техническое примечание

`crypto.randomUUID()` недоступен в небезопасном HTTP-контексте. В проекте есть
fallback `src/lib/id.ts`, но прямые вызовы остались в Academy V2, включая
`CreateCourseModal`, `ExternalAccessPage`, builder, templates и distribution.

### Доказательства

- [Ошибка создания курса](screenshots/01-course-creation-failed.png)
- [Падение публичного training](screenshots/05-public-training-crypto-randomuuid-crash.png)

## QA-ACADEMY-002 — падение подробного отчёта прохождения

Приоритет: P0 / blocker
Маршрут: `/academy/enrollments/:enrollmentId/report`

### Шаги

1. Открыть `/academy/reports`.
2. Выбрать любую существующую строку.
3. Нажать «Отчёт».

### Фактический результат

Error Boundary заменяет экран сообщением:
`Cannot read properties of undefined (reading 'map')`.

### Ожидаемый результат

Открывается подробный отчёт выбранного прохождения либо контролируемое состояние
ошибки API без падения React-экрана.

### Техническое примечание

Ошибка возникает в production bundle `AcademyPlaceholderPage` при вызове `.map`
у отсутствующего поля response DTO.

### Доказательство

- [Падение отчёта](screenshots/02-enrollment-report-crash.png)

## QA-ACADEMY-003 — внутренний плеер падает при открытии назначенного курса

Приоритет: P0 / blocker
Маршрут: `/learn/:enrollmentId`

### Шаги

1. Открыть `/academy`.
2. У курса со статусом «В процессе» нажать «Продолжить обучение».

### Фактический результат

Плеер полностью заменяется Error Boundary:
`Cannot read properties of undefined (reading 'sections')`.

### Ожидаемый результат

Открывается последний доступный урок, структура курса и текущий прогресс.

### Техническое примечание

Ошибка возникает в `InternalEnrollmentPlayerPage`: frontend ожидает поле
`sections`, которого нет в фактическом ответе API.

### Доказательство

- [Падение внутреннего плеера](screenshots/03-course-player-crash.png)

## QA-ACADEMY-004 — реестр внешних учеников падает при открытии

Маршрут: `/academy/learners`

### Шаги

1. Авторизоваться владельцем.
2. Открыть «Академия» → «Внешние ученики».

## Успешные проверки

### Навигация и layout

- Академия V2 включена.
- Роль владельца видит все семь подразделов.
- Контент ограничен `max-width: 1280px`, имеет отступы 24 px.
- На ширине 375 px нет горизонтального overflow страницы.
- Внутренняя навигация прокручивается горизонтально.
- Таблица отчётов имеет отдельный горизонтальный scroll-контейнер.

### Главная и каталог

- Главная показывает 5 назначенных курсов и согласованные счётчики прогресса.
- Продолжаемый курс отображает 25% и `1/4`.
- Каталог корректно показывает пустое состояние.
- Поиск синхронизируется с query-параметром `q`.
- Для поискового запроса отображается отдельное пустое состояние.

### Управление и отчёты

- Пустой список курсов имеет CTA создания и переход к шаблонам.
- Кнопка создания недоступна без названия и активируется после ввода.
- Импорт из базы знаний честно обозначен как недоступный.
- Внутренний отчёт показывает `not started`, `in progress` и `completed`.
- Поиск отчёта сокращает таблицу до одной строки и сохраняется в URL.
- Несуществующий UUID курса показывает штатное состояние «Курс не найден».

### Совместимость ссылок

- `/academy-opus` перенаправляет на `/academy`.
- `/academy-grok/catalog` перенаправляет на `/academy/catalog`.

## Ограничения прогона

Полный сценарий публикации, конструктора, назначения и внешнего распространения
невозможен до исправления QA-ACADEMY-001. Ролевую матрицу нельзя подтвердить
одним аккаунтом владельца. Для успешного внешнего сценария нужен валидный
одноразовый training token.
